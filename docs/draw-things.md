# Draw Things Integration

ImageQueue can generate images locally using the [Draw Things](https://drawthings.ai) CLI. This requires no API key and incurs no per-image cost — everything runs on your Mac.

## Overview

- **macOS only.** The Draw Things column is hidden on Windows and Linux.
- Images are generated by spawning `draw-things-cli` as a subprocess. ImageQueue never communicates directly with the Draw Things GUI app.
- Only **one generation runs at a time** per the Draw Things backend (unlike cloud backends which run several in parallel). This matches the CLI's single-GPU constraint.

---

## Prerequisites

1. Install the [Draw Things app](https://apps.apple.com/app/draw-things-ai-generation/id1559657metadata) from the Mac App Store, or install the CLI standalone.
2. Confirm the CLI is available:
   ```sh
   draw-things-cli --version
   ```
   If the command is not found, locate the binary inside the Draw Things app bundle and add it to your `$PATH`, or configure its full path in Settings (see below).

---

## Configuration (Settings → Draw Things)

| Setting | Default | Description |
|---|---|---|
| `cli_path` | *(empty — uses `draw-things-cli` on `$PATH`)* | Full path to the CLI binary, e.g. `/Applications/Draw Things.app/Contents/MacOS/draw-things-cli` |
| `models_dir` | *(empty — see below)* | Directory where model files are stored |
| `model` | `flux_2_klein_4b_q6p.ckpt` | Default model selected when the column loads |
| `steps` | `4` | Default inference steps |
| `cfg` | `1` | Classifier-free guidance scale |
| `width` / `height` | `1024` | Default output dimensions |
| `seed` | *(null — random)* | Set a fixed seed to reproduce results |
| `negativePrompt` | *(empty)* | Default negative prompt |

---

## Models Directory

The models directory is where model `.ckpt` files are stored and loaded from. ImageQueue resolves it with the following priority:

1. **Configured path** — if `models_dir` is set in Settings, it is used for all operations (list, download, delete, detect, generate).
2. **Draw Things app container** — if `models_dir` is empty and the Draw Things app is installed, ImageQueue automatically finds its container at `~/Library/Containers/com.liuliu.draw-things/Data/Documents/Models`. Model detection and deletion use this path; generation and downloads use the CLI's built-in default (which resolves to the same location).
3. **CLI default** — if neither of the above is available, all CLI operations run without `--models-dir` and the CLI uses its own internal default.

**Recommendation:** leave `models_dir` empty if you use the Draw Things app normally. Only set it if you want to store models in a custom location separate from the app.

## Managing Models

Open the **Manage Draw Things Models** dialog from the hamburger menu (☰ → Manage Draw Things Models).

### Browsing

The dialog lists all models known to the CLI, grouped into downloaded and available-to-download sections.

### Downloading

Two download options are available for each model:

- **Download** — runs in the background within ImageQueue. Progress is not visible but the model becomes available once complete.
- **Terminal** — opens a new Terminal window running the download command. Use this to watch progress or when downloading large models.

### Deleting

The **Delete** button removes the model file from the effective models directory (configured path or Draw Things app container). This cannot be undone.

## Generation

Each generation spawns `draw-things-cli generate` with the following flags derived from the task:

| CLI flag | Source |
|---|---|
| `--model` | Selected model in the column |
| `--prompt` | Prompt text |
| `--output` | Temporary file in the session directory (deleted after reading) |
| `--steps` | Steps setting |
| `--width` / `--height` | Size setting |
| `--seed` | Seed setting (omitted when 0 or null) |
| `--negative-prompt` | Negative prompt (omitted when empty) |
| `--disable-preview` | Always set (suppresses the live preview window) |
| `--models-dir` | Only included when `models_dir` is configured |

## Troubleshooting

**CLI not found**
- Ensure `draw-things-cli` is on your `$PATH`, or set the full path in Settings.
- Test with `draw-things-cli --version` in Terminal.

**No models appear in the column or modal**
- The CLI may not be detecting the models directory. Try configuring `models_dir` explicitly in Settings to the folder containing your `.ckpt` files.

**Generation fails with "model not found"**
- The model file referenced by the task is not present in the models directory the CLI is using. Download it via the Models modal or ensure `models_dir` points to the correct location.

**Generation is slow**
- Draw Things runs on your GPU. Steps, resolution, and model size all affect speed. FLUX Schnell models (the default) are optimised for low step counts (4–8 steps).
